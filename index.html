Game Engine
Equipe

Victor Ivis

Descrição

Uma game engine simples em C++ feita com SDL2. O mapa é carregado usando um arquivo xml que guarda as posições no formato csv. As colisões são lidadas pelo algorítmo AABB e existe a implementação da física para gravidade, atrito, movimentação na diagonal usando seno e cosseno e outros funcionalidades que estão em desenvolvimento. Existe uma versão compilada tanto para windows quanto para linux na aba de tags.

BikeShare Connect
Equipe

Daniel Lima, João Gabriel, Victor Ivis

Descrição

Uma aplicação web mobile para controle de bicicletas coletivas, idealizado para transformar o transporte urbano em uma solução mais inclusiva e sustentável. Para a API REST, foi utilizado TypeScript com o MongoDB como banco de dados e o Redis para otimização das consultas com um cache. No frontend, utilizou-se SCSS para automatização de alguns recursos gráficos. Também é possível visualizar as informações geoespaciais no navegador por charts do Mongo Atlas.

Aplicativo para Controle de Gastos
Equipe

Cauã Pires, Daniel Lima, Victor Ivis

Descrição

Um aplicativo bem simples que permite guardar despesas e lucros para visualizar graficamente a situação financeira do usuário. Atuei no backend fazendo a parte de armazenamento dos dados, que é bastante simples porque não utilizou qualquer banco de dados, mas apenas o próprio local storage. Também fiz toda a parte de geração dos gráficos usando a biblioteca Chart.js e ainda auxiliei nos pop-ups que foram feitos com a biblioteca SweetAlert2.

Prometheus
Equipe

Luiz Fernando, José Anderson, Victor Ivis

Descrição

Uma aplicação REST para gerenciamento e divulgação de eventos. Permite que os moderadores vejam os inscritos e gerem certifados ao fim das atividades para os usuários que participaram. Atuei na parte de desenvolvimento das funções de CRUD na API utilizando o framework Spring Boot. Também ajudei a conectar os endpoints no front-end em React e fiz alguns ajustes nos formulários para envio das informações ao banco de dados PostgreSQL.

Lei de Zipf
Equipe

Victor Ivis

Descrição

Uma apresentação feita com o Jupyter Lab utilizando a biblioteca MatPlotLib do python para os gráficos da distribuição da lei de zipf em diversos livros como Alice no País das Maravilhas, Críton, Dom Casmurro e outros. Essa lei diz que a 2ª palavra mais frequente aparece metade das vezes que a primeira, a 3ª aparece um terço da primeira e assim por diante. Esse conceito tem diversas aplicações na área da estatística como detecção de plágios, processamento de linguagem natural e até previsão do comportamento de populações.
